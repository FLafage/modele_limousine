Deuxième étape : importation des bases de données

Concernant les prix des carcasses en entrée d’abattoir, je me suis appuyé sur les bases de données de FranceAgriMer, accessibles à ce lien : https://visionet.franceagrimer.fr/Pages/SeriesChronologiques.aspx?menuurl=SeriesChronologiques/productions%20animales/viandes/gros%20bovins%20entr%C3%A9e%20abattoir

Un seul jeu de données a été traité pour le prix des carcasses en entrée d’abattoir :

L’historique des cotations de viandes de Limousines en entrée d’abattoir, standards (STD) et sous signes officiels de qualité (SIQO), hors agriculture biologique, au niveau national.

L’import direct de la base n’était pas possible, car le fichier original était au format Excel (.xlsx) avec des cellules fusionnées, ce qui le rendait inutilisable directement sous R. J’ai donc dû retravailler manuellement la base pour obtenir un format exploitable.

Analyse : cette étape est indispensable pour garantir la fiabilité des données. Les fichiers Excel provenant de sources officielles contiennent souvent des mises en forme qui compliquent l’import automatique, et il est préférable de corriger les anomalies avant toute modélisation.

PRIX_CARCASSE_LIMOUSINE <- read.xlsx("C:/Users/faust/Desktop/Modele_Limousine/Limousine_Siqo.xlsx", sheet = "Siqo", detectDates = TRUE)

Troisième étape : traitement de la base de donnée

La base a été nettoyée et restructurée afin de :

  • Convertir les dates en format Date utilisable par R.

  • Transformer les cotations en valeurs numériques, en traitant les éventuels NA ou points manquants.

  • Renommer les colonnes pour plus de clarté.

  • Passer d’un format large (colonnes par type de carcasse) à un format long (une ligne par date et type de carcasse).

  • Calculer le prix moyen par mois et par type de carcasse.

Analyse : ce traitement permet d’obtenir un jeu de données homogène et exploitable pour les modèles statistiques, et réduit le risque d’erreurs lors du calcul des moyennes ou de la visualisation.

CARCASSE_LIMOUSINE <- PRIX_CARCASSE_LIMOUSINE %>%
  mutate(Date = as.Date(Date, origin = "1899-12-30"),
         Annee = year(Date),
         Mois = month(Date),
         Jour = day(Date)) %>%
  mutate(across(c(`U+`, `U=`, `U-`, `R+`), ~ as.numeric(ifelse(. == ".", NA, .)))) %>%
  rename(
    U_plus = `U+`,
    U_egal = `U=`,
    U_moins = `U-`,
    R_plus = `R+`
  ) %>%
  pivot_longer(cols = c(U_plus, U_egal, U_moins, R_plus),
               names_to = "Type_Carcasse",
               values_to = "Prix") %>%
  group_by(Annee, Mois, Type_Carcasse) %>%
  summarise(Prix = mean(Prix, na.rm = TRUE), .groups = "drop") %>%
  filter(!is.na(Prix) & Type_Carcasse != "U_plus") %>% 
  mutate(Date_mois = as.Date(paste(Annee, Mois, "01", sep = "-")))

Observation : Le graphique de visualisation montre l’évolution des prix par type de carcasse. Les lignes pleines représentent les observations réelles, tandis que les lignes en pointillés indiquent la tendance estimée par la régression linéaire.

Cinquième étape : Prédictions jusqu’à 2027

Regression linéaire

Application de du modèle de regression linéaire à tous les types de carcasse

Les modèles ont été appliqués séparément pour chaque type de carcasse (R_plus, U_egal, U_moins) afin d’obtenir des prédictions adaptées à chaque catégorie.

dernier_mois <- max(CARCASSE_LIMOUSINE$Date_mois)

dates_future <- seq(
from = dernier_mois %m+% months(1),  # mois suivant le dernier disponible
to   = as.Date("2027-12-01"),
by   = "month"
)

data_R_plus  <- CARCASSE_LIMOUSINE %>% filter(Type_Carcasse == "R_plus")
data_U_egal  <- CARCASSE_LIMOUSINE %>% filter(Type_Carcasse == "U_egal")
data_U_moins <- CARCASSE_LIMOUSINE %>% filter(Type_Carcasse == "U_moins")

reg_R_plus  <- lm(Prix ~ Date_mois, data = data_R_plus)
reg_U_egal  <- lm(Prix ~ Date_mois, data = data_U_egal)
reg_U_moins <- lm(Prix ~ Date_mois, data = data_U_moins)

creer_pred <- function(df_type, model, nom_type, dates_future) {
  data.frame(
    Annee = year(dates_future),
    Mois = month(dates_future),
    Jour = day(dates_future),
    Type_Carcasse = nom_type,
    Prix = predict(model, newdata = data.frame(Date_mois = dates_future)),
    Date_mois = dates_future
  )
}

pred_R_plus  <- creer_pred(data_R_plus, reg_R_plus, "R_plus", dates_future)
pred_U_egal  <- creer_pred(data_U_egal, reg_U_egal, "U_egal", dates_future)
pred_U_moins <- creer_pred(data_U_moins, reg_U_moins, "U_moins", dates_future)

CARCASSE_LIMOUSINE_COMPLETE <- bind_rows(
  CARCASSE_LIMOUSINE,
  pred_R_plus,
  pred_U_egal,
  pred_U_moins
)

Analyse : L’intégration des prévisions dans le même format que la base historique permet de combiner facilement les données et de visualiser les tendances futures sur le même graphique. Cette approche facilite la comparaison entre les valeurs observées et les valeurs prédites.

Observation sur les résultats : Le modèle de régression linéaire n’est clairement pas performant pour prédire le prix futur des carcasses. La tendance linéaire ne capture ni les variations saisonnières ni les fluctuations ponctuelles du marché. Les prédictions jusqu’en 2027 doivent donc être interprétées avec prudence : elles donnent une estimation approximative de la tendance générale, mais ne permettent pas une prédiction exacte du prix moyen mensuel.

Conclusion : Pour améliorer la précision des prévisions, il serait pertinent d’explorer des modèles plus avancés tels que les modèles ARIMA ou les modèles de séries temporelles avec composantes saisonnières, qui peuvent mieux capter les cycles de prix et les variations spécifiques à chaque type de carcasse.

Arima

Application du modèle de prédiction

prevoir_arima <- function(df_type, nom_type, h) {
  # Créer une série temporelle mensuelle
  ts_data <- ts(df_type$Prix, start = c(min(df_type$Annee), min(df_type$Mois)), frequency = 12)
  
  # Ajuster le modèle ARIMA automatiquement
  fit <- auto.arima(ts_data)
  
  # Prédiction pour h mois
  fc <- forecast(fit, h = h)
  
  # Créer un data.frame avec les prévisions et les dates futures
  data.frame(
    Date_mois = seq(from = max(df_type$Date_mois) %m+% months(1), by = "month", length.out = h),
    Prix = as.numeric(fc$mean),
    Type_Carcasse = nom_type
  )
}

# Nombre de mois à prévoir jusqu'à décembre 2027
h <- length(dates_future)

# Prévisions ARIMA par type
arima_R_plus  <- prevoir_arima(data_R_plus, "R_plus", h)
arima_U_egal  <- prevoir_arima(data_U_egal, "U_egal", h)
arima_U_moins <- prevoir_arima(data_U_moins, "U_moins", h)

# Combiner les prévisions ARIMA
ARIMA_PRED <- bind_rows(arima_R_plus, arima_U_egal, arima_U_moins)
